Thought for 8s
For /compress-pdf-to-5mb, the best answer is: don’t buy an API first. Use a PDF processing library.

What to use
Primary choice: Ghostscript (pdfwrite)
It’s the standard workhorse for PDF compression/rewrite pipelines and is built exactly for generating optimized output PDFs. 

Helpful companion: qpdf
Use it before/after compression for cleanup, structural optimization, and related PDF transforms. qpdf’s docs explicitly include file-size optimization and writing transformed output files. 

Important reality
There is no magic API that guarantees “exactly 5 MB” in one shot.
You usually need an iterative compression loop:

upload PDF

run Ghostscript with a compression profile

check output size

if still above 5 MB, rerun with stronger settings

stop when file is under target or quality floor is reached

That’s the right way to build /compress-pdf-to-5mb.

Best stack for you
Ghostscript for actual compression/rewrite 

qpdf for cleanup/optimization steps around the pipeline 

PDFium only if you later need deeper rendering/viewing logic; it’s not the first choice for this specific feature. PDFium’s official docs emphasize loading/parsing/rasterizing rather than being your main “compress to target size” engine.
